Description
======================================
This email is claiming to be from "Adam Barry", and it is inviting the recipient to the wedding of the sender.

It claims that a short survey has been attached for your RSVP and meal preferences to assist the planning of their wedding.

This spear phishing email uses warm personal language to social-engineer the recipient into opening the attached document "AR_Wedding_RSVP.docm". 

The attachment is a macro-weaponized Word document containing an auto-executing VBA macro (Document_Open) that silently downloads and executes a trojan from GitHub (hxxps[://]github[.]com/TCWUS/Pastebin-Uploader[.]exe) using Microsoft.XMLHTTP and WScript.Shell objects, classified by VirusTotal as a downloader trojan.


Artifact Analysis
======================================
Sender Analysis:
The sender address "abarry@live.com" passed all email authentication checks — DKIM (selector1, live.com), SPF (2a01:111:f403:2c14::801 designated as permitted sender), ARC, and DMARC (p=NONE) — as confirmed by extracted email headers and PhishTool analysis. 

The originating IP resolves via rDNS to mail-bn8nam11olkn20801.outbound.protection.outlook.com, confirming the email was sent through Microsoft legitimate Outlook/Live.com infrastructure. 

DomainTools WHOIS confirms live.com is owned by Microsoft Corporation (registered 1994-12-28, ASN AS8068, Redmond WA). 

The attacker used a real Microsoft Live account rather than a spoofed address, allowing the email to bypass all gateway-level authentication filters.


Attachment Analysis:
The attachment "AR_Wedding_RSVP.docm" was analyzed using oledump.py and VirusTotal. 

Hashes — MD5: 590d3c98cb5e61ea3e4226639d5623d7 | SHA-1: 91091f8e95909e0bc83852eec7cac4c04e1a57c3 | SHA-256: 41c3dd4e9f794d53c212398891931760de469321e4c5d04be719d5485ed8f53e. OLE dump revealed 5 VBA streams including VBA/ThisDocument with a Document_Open() macro that creates three COM objects (Microsoft.XMLHTTP, ADODB.Stream, WScript.Shell) to download hxxps[://]github[.]com/TCWUS/Pastebin-Uploader[.]exe), save it as shost.exe, and execute it silently. 

VirusTotal flagged the file 46/66 vendors with behavioral tags: auto-open, run-file, calls-wmi, create-ole, macros, exe-pattern, write-file, download. Threat family: downloader.autdwnlmer/w97m/o97m. 

PhishTool OLE analysis confirmed active macro (flagged red).


Verdict
======================================
This is a phishing attempt which is of Critical Severity / Malicious Macro Trojan Downloader. 

The email is a targeted spear-phishing attack using social engineering (fake wedding invitation) to deliver a macro-weaponized Word document. 

Upon opening, the Document_Open macro auto-executes without user interaction, downloads a trojan executable from GitHub, and runs it silently on the host. 

All email authentication checks pass because the attacker leveraged a legitimate Microsoft Live account, making this a high-evasion campaign. 

VirusTotal detection of 46/66 confirms the payload is a well-known trojan downloader (w97m/o97m family) with a community score of -7.


Defense Actions
======================================
Immediately quarantine and delete the email and attachment across all mailboxes.

Block the SHA-256 hash ("41c3dd4e9f794d53c212398891931760de469321e4c5d04be719d5485ed8f53e") at the EDR/AV layer.

Block C2 URL "hxxps[://]github[.]com/TCWUS/Pastebin-Uploader[.]exe)" and GitHub account "TCWUS" at the web proxy and firewall.

If the attachment was opened, isolate the endpoint immediately and conduct forensic triage for "shost.exe" execution artifacts. 

Disable macro execution in Microsoft Office via Group Policy (allow only digitally signed macros).

Report "abarry@live.com" to Microsoft abuse at abuse@microsoft.com, and conduct targeted user awareness training on social-engineering lures disguised as personal correspondence.
